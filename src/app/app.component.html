A "toast" is a popup that only stays for a short amount of time. It has the advantage that the user doesn't have to click anything to acknowledge it, so it's much more workflow friendly than a modal popup that blocks the user from doing anything else until they acknowledge it.

Traditionally toasts are shown in a corner of the screen so they're visible but not obnoxiously obtrusive.

Tip: Don't use toasts for long messages or lists of errors where it will take time for the user to read and acknowledge. Toasts should be short and sweet!
